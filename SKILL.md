# 质量管理表单生成（现场数据收集表）

> 本技能生成的是 **SIP / SOP / CP 的下一级执行文件**——现场用来"填数据、做记录"的空白表，不是指导文件本身。

## ① 角色定位

本技能服务的是"需要一张规范表格去现场填数据"的人，而非写指导书的人：

- **现场检验员 / 班组长 / 质量管理员**：需要检验记录表（首检/巡检/终检）去现场记录实测值与判定。
- **试验 / 测试工程师**：需要试验 / 测试报告表，交给试验员填写原始数据与结论。
- **设备点检员 / 设备管理员**：需要设备点检表（班/日/周）去现场打勾记录。

使用者有现场经验，但缺一张字段规范、带签字栏、能直接打印下发的空白表。本技能替其完成"结构标准化 + 字段完整 + 格式可用"，实质项目与标准仍由使用者按 SIP/SOP/CP 提供。

## ② 使用场景

| 触发条件 | 手头材料 | 典型诉求 |
|----------|----------|----------|
| 现场要开始一项检验 / 试验 / 点检，缺记录表 | 产品/零件信息、检验项目清单（来自 SIP 的下级要求） | "给我一张 XX 产品首件检验记录表" |
| 旧记录表字段不全、没签字栏、无法归档 | 现有表样（可投喂） | "按这个重做一张规范的" |
| 设备新增加点检要求 | 设备编号、点检项目 | "出一张日常点检表" |
| 出货 / 来料要做试验报告 | 试验项目、标准 | "做个试验报告表模板" |

触发关键词：检验记录表、试验报告表、测试记录表、点检表、现场填的表、记录表单模板。

## ③ 解决的问题

- **混淆层级**：SIP/SOP/CP 是"怎么做的指导文件"（由对应技能生成）；本技能产出的是"做完填什么的记录表"——是它的下一级执行文件。过去把它们混在一起导致产出物定位不清。
- **输出格式错配**：现场要的是能打印、能下发的 Excel/Word 空白表，不是网页报告。本技能只出 **Excel(.xlsx)** 与 **Word(.docx)**。
- **字段不全难归档**：自动补齐表头信息区、签字栏，判定/结果列带下拉（合格/不合格、正常/异常），到现场直接填、填完能归档。

## ④ 能力边界

**能做**
- 按三类分支生成标准化空白表：检验记录表 / 试验测试报告表 / 点检表。
- 自动补齐表头信息区（产品/零件/批次/依据/日期/班次等）、字段表头、签字栏（编制/审核/日期）。
- 判定/结果列内置下拉（合格/不合格 或 正常/异常），减少现场填错。
- 输出 Excel(.xlsx) 与 Word(.docx) 模板，含列宽与打印设置，可直接下发打印。
- 调用 `scripts/build_form.py` 渲染。

**不做**
- 不生成 SIP / SOP / CP 本身（那是指导文件，由对应技能负责）。
- 不替代专业技术判断：检验项目、标准、抽样方案由使用者按图纸/客户/SIP 提供并确认（标注"★待确认"）。
- 不编造标准号；不确定处标注「待企业补充」。
- 不强制交互锁：信息不全以合理默认+标注处理，不反复追问。

**输入要求**
- 至少给"表单类型（检验/试验/点检）"；其余信息越全越准，缺省用默认值+标注。
- 可选：`--meta` 提供表头字段（产品名称、零件号、设备编号等）。

## ⑤ 交互与角色设定

- **语气**：务实、偏现场口语，避免空话；用"填、打勾、签字、归档"等词。
- **采集方式**：先定类型（3 类分支），再一次性问关键表头；不强制追问，默认填充+标注。
- **是否强制交互锁**：否（生成为主，非分析规划类）。
- **引导深度**：中等，给字段模板与默认，标需确认处。
- **输出确认**：产出 xlsx/docx 后，提示核对实质内容（尤其标准、抽样）。

## ⑥ 处理流程

1. **确认类型**：检验记录表 / 试验测试报告表 / 点检表。
2. **收集表头信息**（一次性）：产品/零件/设备、依据文件编号、日期/班次等。
3. **按类型生成空白表结构**（字段见 ⑦）。
4. **加判定/结果下拉、签字栏、打印设置**。
5. **渲染**：
   ```bash
   python scripts/build_form.py --type inspection --format both --rows 15 \
       --title "XX产品首件检验记录表" \
       --meta "产品名称=XX" "零件号=ABC-001" "检验阶段(首检/巡检/终检)=首检"
   ```
   不传参时用内置样本跑通，产出三类演示模板（xlsx+docx）。

## ⑦ 输入输出

**输入**：表单类型 + 表头信息（文本 / `--meta`）。

**输出**：Excel(.xlsx) 和 / 或 Word(.docx) 空白模板，置于**用户当前工作目录（运行技能时所在目录）**。均为**带空白数据行的可填写模板**，非报告。技能目录本身不含 `outputs/`（符合技能发布规范）。

**三类表单字段定义**

| 类型 | 表头信息区 | 字段列 | 下拉 |
|------|-----------|--------|------|
| 检验记录表 | 表单名称 / 产品名称型号 / 零件图号 / 批次订单 / 检验阶段 / 检验依据(SIP) / 日期 / 班次 | 序号, 检验项目, 标准要求, 实测值, 单位, 判定(合格/不合格), 不良现象备注, 检验员 | 判定：合格/不合格 |
| 试验测试报告表 | 表单名称 / 试验名称 / 样品编号 / 样品状态 / 试验设备 / 环境(温湿度) / 依据标准 / 日期 | 序号, 试验项目, 试验条件方法, 技术要求, 测试结果, 单位, 结论(合格/不合格), 偏离说明, 测试人, 复核人 | 结论：合格/不合格 |
| 点检表 | 表单名称 / 设备名称 / 设备编号 / 点检部位 / 点检周期 / 日期 / 班次 | 序号, 点检项目, 点检标准要求, 点检方法, 结果(正常/异常), 异常处理措施, 点检人, 确认人 | 结果：正常/异常 |

## ⑧ 内置知识 / 参考

- **表单字段详表**：见 `references/form-templates.md`（三类表单的字段定义、下拉项、适用场景与样例）——本技能的知识参考文件。
- **表单层级**：SIP/SOP/CP（指导文件，上级）→ 本技能记录/报告/点检表（执行文件，下一级）。
- **编号建议**：QR（质量记录）前缀，如 `QR-QI-<流水>`；企业自有规则以使用者为准。
- **判定术语**：合格/不合格（检验/试验）、正常/异常（点检）。
- **抽样/标准**：GB/T 2828.1、图纸、客户 SIP——由使用者提供，未提供标「待企业补充」。
